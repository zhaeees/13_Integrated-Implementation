## 학생관리 서버 연계 예제 프로젝트 

### 프로젝트 개요 
 외부 시스템에서 제공되는 데이터를 내부 웹 프로그램과 연결하여 웹화면에 구현하는 예제입니다. 
 
### 파일구성 
 - `index.html`       /* 화면 구조 */
 - `style.css`        /* UI 스타일 */
 - `script.js`        /* 메인 로직 */
 - `server-data.js`   /* 외부 학생 관리 서버에서 제공하는 학생 데이터 */

### 실행 절차
1. 프로젝트 파일 전체를 다운로드합니다.
2. 아래 파일들이 같은 폴더에 있는지 확인합니다. 
  - index.html
  - style.css
  - script.js
  - server-data.js
3. `index.html` 파일을 브라우저(Chrome 권장)에서 실행합니다.
4. 학생 데이터가 화면에 정상적으로 출력되는지 확인합니다.

# 시험 요구사항 설명

## 외부 서버는 무엇인가?
외부 서버는 현재 웹 페이지`index.html`와는 다른 위치에서 데이터를 제공하는 시스템을 의미합니다. 주로 데이터베이스와 연결되어 있으며, 요청(Request)에 따라 필요한 데이터를 응답(Response)으로 전달합니다.
실제환경에서는 사용자 정보, 학생 목록, 상품 정보, 예약, 주문, 상태 데이터 등의 데이터들을 외부서버에서 받아옵니다.
이 프로젝트에서는 실제 서버를 사용하지 않고, `server-data.js` 파일을 외부 서버 역할로 가정하여 구현하였습니다.
실제 서버 대신 서버에서 내려오는 데이터 구조를 미리 정의하였고 html에서 <script src="server-data.js"></script> 태그를 통해 파일을 불러와 시물레이션 한것입니다.

## 어떤 데이터가 연계데이터인가?
연계 데이터란  
**외부 서버(또는 서버 역할의 시스템)에서 전달되어, 프론트엔드 화면을 구성하는 데 사용되는 데이터**를 의미합니다.
즉, 서버에서 내려온다고 가정하고 사용하는 데이터를 연계 데이터라고 합니다.

본 프로젝트에서의 연계 데이터는 `server-data.js`에 정의된 **학생 데이터 전체**입니다.
 학생 ID, 이름, 학년 반,상태 (출석 / 지각 / 결석)
이 데이터들은 HTML 파일에 직접 작성되지 않으며,  **JavaScript를 통해서만 전달받아 화면에 출력**됩니다.  

   👉 본 프로젝트에서 연계 데이터란,  `server-data.js`를 통해 **외부 서버에서 전달된 것으로 가정한 학생 정보 데이터**를 의미합니다.


## 외부서버 -> 내부 웹 데이터 흐름
이 예제는 **외부 서버에서 전달된 데이터를 내부 웹 화면에 표시하는 전체 데이터 흐름**을 학습하기 위한 구조입니다.  
데이터의 흐름은 다음과 같습니다. 

**1단계** ***외부 서버에서 데이터 제공***
외부 서버는 학생 정보를 **데이터 형태로 제공**합니다.   
학생 목록,학생 고유 정보 (ID, 이름, 학년 등),화면 상태를 결정하는 값을 제공  
  👉 `server-data.js`가 **외부 서버 역할**을 수행합니다.

**2단계** ***내부 웹에서 데이터 수신***
`script.js`는 외부 서버로부터 데이터를 **수신하는 역할**을 합니다.  
HTML은 데이터를 직접 알지 못함, JavaScript만 데이터에 접근, 데이터는 **변수(상태)** 형태로 관리됨  
  👉 이 단계가 **서버 연계의 핵심 구간**입니다.
   
**3단계** ***데이터 가공 및 처리***
수신된 데이터는 그대로 화면에 출력되지 않고, 화면 구조에 맞게 **가공 및 처리**됩니다.  
반복 처리,조건 처리,화면 출력용 데이터 변환  
  👉 이 단계에서는 **데이터 중심 사고**가 요구됩니다.

**4단계** ***HTML 화면에 출력***
가공된 데이터는 DOM 조작을 통해 화면에 렌더링됩니다.  
데이터 변경 시 화면 자동 반영, HTML 구조는 유지, 화면의 내용만 데이터에 의해 변경됨  
  👉 **데이터가 화면을 지배하는 구조**입니다.  

본 프로젝트는 외부 서버에서 전달된 데이터를 내부 웹 애플리케이션에서 처리하여 화면에 출력하는 전체 데이터 흐름을 학습하기 위한 예제입니다.

## 왜 이구조가 "연계"인지
연계란 단순히 데이터를 사용하는 것이 아니라,  
**서로 다른 역할을 가진 시스템들이 데이터를 기준으로 연결되어 동작하는 구조**를 의미합니다.
본 프로젝트는 다음의 연계 구조 조건을 충족합니다.

### 연계 구조의 핵심 조건
1. **데이터의 출처가 화면 밖에 있음**       →  화면과 데이터가 분리됨  
2. **데이터가 중간 매개체를 통해 전달됨**   →  JavaScript가 연결자(Connector) 역할 수행  
3. **데이터 변경이 화면에 영향을 줌**       →  화면이 데이터를 기준으로 동작  
4. **화면이 데이터 구조에 의존함**          →  주도권이 데이터에 있음  

이 프로젝트는  외부 서버 역할의 데이터, 내부 웹 로직, 화면 출력이  **분리되어 있으면서도 데이터 기준으로 연결되어 함께 동작**합니다.
따라서 본 프로젝트는  
**외부 시스템과 내부 웹의 데이터 연계 구조**를 기반으로 구현되었습니다.

## 이 예제로 평가 가능한 능력단위
이 프로젝트는 단순한 화면 구현 예제가 아니라,  
**데이터 연계 기반 프론트엔드 역량을 평가할 수 있는 구조**입니다.

 ***1. 외부 데이터 연계 이해 능력***
- `server-data.js`를 외부 서버 역할로 분리
- HTML에 데이터 직접 작성하지 않음  
 👉 JavaScript를 통해서만 데이터 접근

***2. 데이터 중심 사고 능력***
- 학생 목록이 데이터 구조에 의해 결정
- 화면 구조는 유지하고 내용만 데이터로 변경  
 👉 정적 화면이 아닌 동적 UI 구현

***3. 역할 분리 설계 능력***
- `index.html` : 화면 구조
- `script.js` : 데이터 처리 및 화면 제어
- `server-data.js` : 데이터 제공  
 👉 파일별 역할이 명확한 기본 아키텍처 구조

***4. JavaScript 데이터 처리 능력***
- 배열 / 객체 구조 이해
- 학생 데이터 반복 렌더링
- 조건에 따른 화면 출력 제어
- 데이터 상태 기반 DOM 조작  
 👉 실무형 JS 기초 역량

***5. 확장 가능성 고려 능력***
- `fetch` 기반 실제 서버 연계로 전환 가능
- 필터 / 검색 / 상세보기 기능 확장 가능
- 서버 교체 시 화면 코드 재사용 가능  
 👉단기 과제가 아닌 서비스 관점 사고

**데이터 흐름, 역할 분리, JavaScript 기반 화면 제어 능력**을  종합적으로 평가할 수 있는 예제 프로젝트입니다.
